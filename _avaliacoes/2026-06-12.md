## Problemas encontrados
1. **Contraste (WCAG AA)** — texto branco 13px falha em `.pill--vermelho` (#ED2647 → 4.22:1) e `.pill--verde` (#3FA110 → 3.33:1). Azul (7.48:1) e laranja c/ preto (7.22:1) passam. Correções possíveis: texto preto nessas pills, ou tons mais escuros (`#C71B38`, `#2E7A0C`).
2. **Links mortos no footer** — Behance/Instagram/LinkedIn/ArtStation apontam para `#` (todas as páginas). `rel="noopener"` sem `target="_blank"` é inócuo. Precisa das URLs reais.
3. **`pt / en` é um `<span>` decorativo** — parece interativo mas não é (falsa affordance). Remover até existir versão EN, ou marcar visualmente como inativo.
4. **Filtros sem estado acessível** — `.filter-btn` não usa `aria-pressed`; leitores de tela não sabem qual filtro está ativo nem que a lista mudou (sugestão: `aria-pressed` + região `aria-live` com contagem de resultados).
5. **`scroll-behavior: smooth` global** — fora do guard de `prefers-reduced-motion` (o guard atual cobre só `.reveal` e marquee).
6. **Cards de projeto não são clicáveis** — não há páginas de detalhe; o portfólio não mostra o trabalho em si (thumbs são placeholders de texto). É a maior lacuna de conteúdo do site.
7. **Arquivos órfãos no repositório** — `js/three-scene.js`, `css/publica.css`, `files/clean_css.js`, `files.zip`, `run.ps1` não são referenciados por nenhum HTML; `assets/images/Motion.glb` sugere uma cena 3D planejada. Confirmar intenção antes de qualquer limpeza.
8. **Sem foco visível customizado** — só inputs têm `outline` custom; links/botões dependem do default do navegador (funciona, mas um `:focus-visible` consistente com o azul reforçaria o design system).
9. **Metadados sociais ausentes** — sem Open Graph/Twitter cards; ao compartilhar o portfólio o link aparece sem preview (relevante para divulgação).

## Propostas de motion (aguardando aprovação no FEEDBACK.md)

### Proposta 1 — Reveal em cascata (stagger) nos grids
Fundamento: padrão editorial visto no estudo (galerias curadas); dá ritmo de leitura sem custo de performance. Os cards de um mesmo grid hoje aparecem todos juntos.
```css
/* style.css — após o bloco .reveal */
.areas-grid .reveal:nth-child(2),
.projects-grid .reveal:nth-child(2) { transition-delay: .12s; }
.areas-grid .reveal:nth-child(3),
.projects-grid .reveal:nth-child(3) { transition-delay: .24s; }
@media (prefers-reduced-motion: reduce) {
  .areas-grid .reveal, .projects-grid .reveal { transition-delay: 0s; }
}
```

### Proposta 2 — Pausa do marquee em hover/focus
Fundamento: WCAG 2.2.2 (conteúdo em movimento deve poder ser pausado) + legibilidade. Custo zero.
```css
/* style.css — junto do bloco .marquee */
.marquee:hover .marquee-track,
.marquee:focus-within .marquee-track { animation-play-state: paused; }
```

### Proposta 3 — Hover informativo nos cards de projeto
Fundamento: Site of Sites (hover revela informação/affordance). Uma seta "→" desliza no título ao passar o mouse, sinalizando que o card levará a um detalhe (prepara a futura página de case).
```css
/* style.css — junto de .project-body h3 */
.project-body h3::after {
  content: '→';
  display: inline-block;
  margin-left: 8px;
  opacity: 0;
  transform: translateX(-6px);
  transition: opacity .25s ease, transform .25s ease;
  color: var(--azul);
}
.project-card:hover .project-body h3::after,
.project-card:focus-within .project-body h3::after { opacity: 1; transform: none; }
@media (prefers-reduced-motion: reduce) {
  .project-body h3::after { transition: none; }
}
```

### Correção recomendada (não é motion): contraste das pills
```css
.pill--vermelho { background: #C71B38; } /* 5.6:1 com branco */
.pill--verde { background: #2E7A0C; color: var(--branco); } /* ~5.9:1 */
/* alternativa sem mudar a paleta: texto preto, como na pill laranja */
```
